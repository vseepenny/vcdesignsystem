VSee targets WCAG 2.1 AA. These patterns ensure every user — including those using keyboards, screen readers, or assistive tech — can navigate the product.

Key Principles
⌨️
Keyboard navigable: All interactive elements must be reachable and operable via keyboard alone. Use correct semantic HTML (<button>, not <div onclick>).
🏷️
Labelled controls: Every input, select, and textarea must have a visible <label> or an aria-label. Icon-only buttons need aria-label.
🔊
Screen reader support: Use ARIA roles for modals (role="dialog"), live regions (aria-live) for toasts, and aria-expanded for accordions and dropdowns.
